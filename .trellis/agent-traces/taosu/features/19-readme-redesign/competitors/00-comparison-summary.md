## Key Patterns to Adopt

### Must Have
1. **5-6 word slogan** - Clear positioning
2. **1 GIF demo** - Show core workflow
3. **Installation command** - Copy-paste ready
4. **Features with icons** - Scannable
5. **Quick Start (3-5 steps)** - Fast time-to-value
6. **Links to docs** - Don't bloat README

### Nice to Have
1. Multi-language README (Chinese)
2. Multiple install methods
3. Enterprise logos (when available)
4. Testimonials section
5. FAQ in README (like OpenCode)

### Avoid
1. README as complete documentation
2. No visual demo
3. Wall of text without structure
4. Missing installation instructions
5. No community links

# Brainstorm - 需求澄清（面向 AI 编码）

在实现前，与用户协作完成需求澄清，适配 AI 编码流程：

* **任务优先**（先落盘）
* **先行动再提问**（减少低价值问题）
* **技术选择先调研**（不把方案设计压力转嫁给用户）
* **先发散后收敛**（充分考虑后聚焦 MVP）

---

## 适用时机

当用户描述开发任务，尤其满足以下情形时，从 {{CMD_REF:start}} 进入本流程：

* 需求不清晰或持续变化
* 存在多条可行实现路径
* 需要权衡（体验、可靠性、维护性、成本、性能）
* 用户暂时无法直接给出最佳方案

---

## 核心原则（不可跳过）

1. **任务优先（尽早留痕）**：会话开始就确保任务目录存在
2. **先行动再提问**：能从仓库/文档/配置中得出的信息先自行获取
3. **一次只问一个问题**：每次仅问一个高价值问题
4. **优先给具体选项**：决策问题给 2~3 个可执行方案 + 取舍
5. **技术问题先调研**：先调研行业/同类工具，再给建议
6. **先发散后收敛**：充分考虑边界与演进后收敛到 MVP，并明确 out-of-scope
7. **不问元问题**：不要问“要不要我搜一下/贴代码吗”之类问题

---

## 第 0 步：先确保任务存在（必须）

任何问答前先确保有任务；如果没有，立刻创建。

```bash
TASK_DIR=$(python3 ./.trellis/scripts/task.py create "brainstorm: <short goal>" --slug <auto>)
```

立刻创建或补齐 `prd.md`，先写已知信息，后续持续迭代更新。
